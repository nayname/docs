## Core Differences

| Aspect | Cosmos EVM | Standard EVM | Impact |
|--------|------------|--------------|---------|
| **Transaction Ordering** | Arrival time to proposer\*\* | Gas price and nonce | Deterministic vs competitive ordering |
| **Transactions per Block** | One per EOA per block\*\* | Multiple per account | Simplified transaction flow |
| **Mempool** | No app-side mempool\*\* | Traditional mempool | Limited MEV opportunities |
| **Consensus** | Tendermint BFT | Proof of Work/Stake | 1-2 second vs 12+ second blocks |
| **Finality** | Immediate (single block) | Probabilistic (12+ confirmations) | No waiting period vs 12-15 minute finality |
| **Chain Reorganizations** | Not possible | Possible | Event handling simplified |
| **Address Formats** | Dual (0x... and cosmos1...) | Ethereum only (0x...) | Cross-format compatibility |
| **Gas Token** | Native chain token | ETH/derivatives | Chain-specific economics |
| **Base Fee** | Returns 0 | Dynamic based on demand | Different fee calculation |
| **Block Time** | 1-2 seconds | 12-15 seconds (varies by L2) | Faster transaction confirmation |
| **Cross-chain** | Native IBC | Bridge protocols | Built-in interoperability |

<footnote>\*\*This will no longer be the case with the upcoming release of the [app-side mempool](https://www.notion.so/skip-protocol/EXT-Cosmos-EVM-Appside-Mempool-21e8c17818cb80028f28cfa440d7a7e7)</footnote>

## Gas Price Behavior

<CodeGroup>
```solidity Cosmos EVM - BASEFEE Returns 0
// This will return 0 on Cosmos EVM
uint256 baseFee = block.basefee;

// Use this pattern instead for fee estimation
uint256 gasPrice = tx.gasprice;
```

```solidity Standard EVM - BASEFEE Available
// This returns actual base fee
uint256 baseFee = block.basefee;

// Can be used for dynamic fee calculations
uint256 maxFee = baseFee * 2 + priorityFee;
```
</CodeGroup>

## Precompiled Contracts

### Cosmos EVM Precompiles

Cosmos EVM provides access to native Cosmos SDK modules through precompiled contracts:

| Module | Address | Purpose |
|--------|---------|---------|
| Staking | `0x800` | Delegate, undelegate, redelegate tokens |
| Distribution | `0x801` | Claim rewards, manage withdrawals |
| IBC Transfer | `0x802` | Cross-chain token transfers |
| Governance | `0x803` | Submit/vote on proposals |

For detailed usage examples and implementation guides, see [Precompiled Contracts](/docs/documentation/smart-contracts/precompiles).

| `0x03` | `ripemd160` | Yes |

## JSON-RPC Method Support

Cosmos EVM implements most standard Ethereum JSON-RPC methods with some limitations due to consensus differences:

| Method Category | Examples | Notes |
|----------------|----------|-------|
| **✅ Core Methods** | `eth_getBalance`, `eth_call`, `eth_sendTransaction`, `eth_getTransactionReceipt` | Full compatibility |
| **⚠️ Gas Methods** | `eth_gasPrice`, `eth_estimateGas`, `eth_maxPriorityFeePerGas` | EIP-1559 partial - `eth_feeHistory` not implemented |
| **✅ Block Methods** | `eth_blockNumber`, `eth_getBlockByHash`, `eth_getBlockByNumber` | Immediate finality |
| **❌ Mining Methods** | All `miner_*` methods | Not implemented - Tendermint consensus |
| **✅ Uncle Methods** | `eth_getUncleCount*` (always 0), `eth_getUncleBy*` (always null) | Returns expected values for no uncles |
| **⚠️ Debug Methods** | `debug_traceTransaction`, `debug_traceBlockByNumber` | Limited functionality, some issues |
| **❌ Admin Methods** | `admin_*` methods | Not implemented |
| **⚠️ Filter Methods** | `eth_newFilter`, `eth_newBlockFilter` | Filters created but `eth_getFilterChanges` fails |

For a complete list of all JSON-RPC methods with implementation status, see the [Ethereum JSON-RPC Methods](/docs/api-reference/ethereum-json-rpc/methods) reference.

### Additional JSON-RPC Methods

Cosmos EVM implements these non-standard methods:

| Method | Purpose |
|--------|---------|
| `eth_getTransactionLogs` | Retrieve logs for a specific transaction |
| `eth_getBlockReceipts` | Get all receipts for a given block |
| `debug_freeOSMemory` | Force garbage collection |
| `debug_setGCPercent` | Configure garbage collector |
| `debug_memStats` | Return detailed memory statistics |
| `debug_setBlockProfileRate` | Set block profiling rate |
| `debug_writeBlockProfile` | Write block profile to file |
| `debug_writeMemProfile` | Write memory profile to file |
| `debug_writeMutexProfile` | Write mutex profile to file |

### Unimplemented Standard Methods

These standard Ethereum methods are not available:

| Method | Category | Reason |
|--------|----------|--------|
| `eth_getFilterChanges` | Filters | Filter management not implemented |
| `eth_getFilterLogs` | Filters | Filter management not implemented |
| `eth_feeHistory` | EIP-1559 | Fee history tracking not implemented |
| `eth_createAccessList` | EIP-2930 | Access lists not supported |
| `eth_resend` | Transaction Pool | Transaction replacement not supported |
| `eth_pendingTransactions` | Transaction Pool | Pending transaction queries not implemented |
| `personal_sendTransaction` | Personal | Personal transaction sending not implemented |
| `personal_ecRecover` | Personal | Signature recovery not implemented |
| All `miner_*` methods | Mining | No mining with Tendermint consensus |
| All `admin_*` methods | Administration | Node administration not exposed |
| `eth_subscribe` syncing | WebSocket | Only newHeads, logs, and newPendingTransactions events supported |
